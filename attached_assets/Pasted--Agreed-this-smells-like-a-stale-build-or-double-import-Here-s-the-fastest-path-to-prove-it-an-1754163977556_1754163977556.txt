## 4  Guard against this ever happening again

1. **Move build to `dist/` only** – configure `tsconfig` or `next.config.js` so compiled JS never pollutes `src/`.
2. **Add a pre-start script** in `package.json`:

   ```json
   "scripts": {
     "prestart": "rimraf dist && npm run build"
   }
   ```
3. **CI check**: fail the build if any `.js` exists in `src/`.

---

## 5  If logs still don’t fire (rare)

* **BullMQ worker importing server**: confirm worker file doesn’t `require('../server')`.
* **Reverse proxy / edge function**: verify Vercel/Netlify `middleware.ts` or Cloudflare Worker isn’t mounted on `/api/quotes`.
* **Container mix-up**: run `lsof -i :<port>` inside the pod; ensure only one PID owns the HTTP port.
